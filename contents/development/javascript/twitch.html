twitch × Webhook

まずtwitchのWebhookやAPIを使うためには、access token を取得する必要がある。
Getting OAuth Access Tokens
access token にはuser access token と app access tokenの2種類があり、それぞれ用途は以下の通り。

  • user access token：主にtwitch の API を呼び出す際に使用。
  • app access token：主に Webhook を使用する場合に必要。

Twitch API
twitchにおいて、ユーザーを示す項目は大きく3種類あります。

  • user_name/login_name：ログインするときのid。アカウントの設定画面の日本語名称は「ユーザー名」。後から変更できる。
  • display_name：チャット欄などに表示される名前。アカウントの設定画面の日本語名称は「表示名」。後から変更できる。設定していないユーザーもいる。
  • user_id：twitchが裏で採番している一意なid。通常意識しない。変更不可。

呼び方は統一されていない気がしますが、項目としてはこの3つがある。display_nameが設定されている場合、チャットしたときの名前が「あああ(aaa)」と表示されるが、「あああ」がdisplay_nameで「aaa」がuser_nameです。
twitchのAPIを使用するときは user_name と user_id を結構使うので、この違いは把握しておきたい。
下記リンク先から user_name と user_id を相互に変換できる。結構使うことはあるので押さえておきたい。

webhookよりはtmi.jsを使った方が（たぶん）早いので基本的にtmi.jsを使えばいいと思うが、raidはwebhookの方が使い勝手が良いと思います。
tmi.js の raidだと、 display_name を設定しているユーザーの場合、display_name のみが返されるため、 user_name を取得できないからです。display_name をキーに user_name を返してくれるAPI は（たぶん）ないので、結構困ります。
Webhookは両方返してくれる。
tmi.js を改造するブログ記事もありますが、外部ライブラリを改造すると、それが原因で予期せぬ挙動をすることも考えられるので、あまりお勧めできません。

'use strict';
import tmi from 'tmi.js';
const opts = {
  identity: {
      username: acdata.username,
      password: 'oauth:'+acdata.oauth_token,
  },
  channels: [
      targetChannel
  ]
};
const twClientBot = new tmi.client(opts)

// チャットに反応
twClientBot.on('message', (channel, userstate, msg, self) => {
    if (self) return; // Ignore messages from the bot
    // 何か処理
});

// チアーに反応
twClientBot.on('cheer', (channel, userstate, message) => {
  // 何か処理
});

// サブスク（初回）に反応
twClientBot.on('subscription', (channel, username, method, message, userstate) => {
  // 何か処理
});

// サブスク（再サブスク）に反応
twClientBot.on('resub', (channel, username, months, message, userstate, methods) => {
  // 何か処理
});

// サブギフ（くれた人）に反応
twClientBot.on('submysterygift', (channel, username, numbOfSubs, methods, userstate) => {
  // 何か処理
});

// サブギフ（貰った人）に反応
twClientBot.on('subgift', (channel, username, streakMonths, recipient, methods, userstate) => {
  // 何か処理
});

// twitchのサーバに接続したら動く
twClientBot.on('connected', async (addr, port) => {
  // 何か処理
});

// Connect to Twitch:
twClientBot.connect();
